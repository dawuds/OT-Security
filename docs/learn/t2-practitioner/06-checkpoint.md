# Checkpoint — Practitioner deliverables review

> **Tier 2 / Lesson 6 — 30 minutes.**

## What you should now own

By the end of T2 you should physically have these five artefacts:

1. **A passive asset list** (from Lesson 1) — at least 10 devices, each with IP, vendor, protocol seen.
2. **A captured PCAP** (from Lesson 2) — proof you can generate and analyse OT traffic.
3. **A Purdue diagram** (from Lesson 3) — every device labelled with (Level, SL-T) and a one-line justification.
4. **An IDMZ gap list** (from Lesson 4) — every IT-to-OT path that bypasses the IDMZ, each cited to a control.
5. **A remote-access policy draft** (from Lesson 5) — at minimum 4 pages, addressing all seven OT-specific extensions.

If any of these is missing, you skipped the lab. Go back.

## Self-grading rubric

For each artefact, ask:

| | Pass | Fail |
|---|---|---|
| Asset list | Each device has a justified Purdue level | "Don't know" / "TBD" entries |
| PCAP | At least one Modbus FC16 (write) shown | Read-only traffic only |
| Purdue diagram | All segmentation violations are circled | Diagram looks clean — too clean to be true |
| IDMZ gap list | Each gap cites SR + NACSA section | "Should fix" with no citation |
| Remote-access policy | All 7 OT-specific sections present | Generic IT policy with "OT" in the title |

If 4 of 5 pass, you're ready for T3. If fewer, repeat the failing labs.

## What you can now do that you couldn't before

- Run a passive asset discovery without crashing PLCs.
- Read a Modbus capture and tell read from write.
- Look at a network diagram and point to the IDMZ failure mode.
- Defend a remote-access policy in front of a plant manager and an auditor.

## What you still cannot do (T3 will fix)

- Stand up the full 10-pillar B.A.S.I.C. S.T.A.R.T. programme.
- Run a tabletop exercise.
- Produce the Board-pack for a regulatory inspection.
- Map your programme against NACSA s23 audit findings.
