# Checkpoint — Orientation Self-Test

> **Tier 1 / Lesson 5 — 10 minutes.** Eight questions. Answer without looking. If you miss more than two, re-read the relevant lesson before moving on to T2.

## The questions

1. **What is the OT triage order during an active incident, and how does it differ from IT?** *(Lesson 1)*
2. **Why is active vulnerability scanning forbidden in OT environments? Name a specific failure mode.** *(Lesson 1)*
3. **Name the seven Purdue levels in order — including Level 3.5. What lives at each?** *(Lesson 2)*
4. **Define the protocol break, and explain where it must occur in the IDMZ pattern.** *(Lesson 2)*
5. **What is a Safety Instrumented System (SIS), why is its isolation from BPCS the cardinal rule, and which incident violated this principle?** *(Lessons 2 + 3)*
6. **For each of the five canonical incidents, name (a) the initial access vector and (b) the OT impact:** *(Lesson 3)*
   - Stuxnet
   - Ukraine 2015
   - TRITON
   - Colonial Pipeline
   - Oldsmar
7. **Name the seven IEC 62443 Foundational Requirements (FRs) — what does each cover?** *(Lesson 4)*
8. **For an NCII operator in Malaysia: which NACSA Act 854 section requires the security audit, and which section sets the 6-hour incident notification clock?** *(Lesson 4)*

## Self-grading

- **8/8** — You're ready for T2. Move on.
- **6–7/8** — Re-read the lesson(s) you missed; come back tomorrow.
- **≤ 5/8** — Re-read the whole tier. T2 builds on this; it will not work otherwise.

## Optional stretch

If you enjoyed the questions, try these — they are *T2-level* and will be answered later:

- Given a network capture, how would you produce a Purdue-mapped asset inventory **without** active scanning?
- For a chlorination dosing PLC, what compensating control would let you mitigate a CVSS-9.8 vulnerability you cannot patch?
- For an external vendor needing remote PLC access, what does a NACSA-defensible access pattern look like?
